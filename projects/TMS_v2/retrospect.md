📄 TMS – AI 기반 테스트 관리 시스템

비전공자 QA가 Cursor와 함께 1주일 만에 만든 TestRail 대체 도구

🚀 프로젝트 소개

TMS(Test Management System)는 테스트 케이스를 보다 구조적으로 관리하기 위해 만든
AI 기반의 웹 테스트 관리 도구입니다.

기존 회사에서는 테스트 케이스를 구글 스프레드시트로 관리했는데,

버전 충돌

구조 붕괴

협업 어려움

검색/정렬/이력 관리 한계

등의 문제가 반복되었습니다.

저렴한 테스트 도구들을 사용해 보았지만 팀에 완전히 맞는 제품은 없었고,
결국 **“직접 만들자”**라는 결론에 이르렀습니다.

이 프로젝트는 코딩 경험이 거의 없던 QA가 Cursor와 함께 단 1주일 만에 MVP를 완성한 실전 사례입니다.

🧱 기술 스택 (AI 추천 기반)

Cursor AI가 추천한 스택을 그대로 사용했습니다.

Frontend: React

Backend: Node.js + Express

DB: MySQL

Auth: JWT

Infra: GitHub + Vercel(또는 로컬 환경)

모든 구현은 AI 주도 + 사람(나)의 명확한 기능 설계 방식으로 진행되었습니다.

🗂 핵심 기능
🔐 1. 로그인 및 권한 관리

회원가입

관리자 승인 후 로그인 가능

JWT 기반 인증

관리자 기능(가입 승인, 비밀번호 초기화)

📁 2. 테스트 케이스 관리

계층형 폴더 구조

드래그 앤 드롭 이동 및 정렬

Rich Text Editor 기반 케이스 작성

번호 리스트/불렛 리스트 지원(사용자 요청 기반 구현)

📥 3. CSV Import

Import 과정에서 폴더 자동 생성 가능

선택한 폴더 기반 플랜 생성

진행 중 플랜 실시간 업데이트

개별 실행 / 벌크 실행 지원

메모 입력 시 URL 자동 링크 처리

💡 내가 실제로 AI에게 사용한 프롬프트 예시
✔ 케이스 작성 기능 개선 요청(리스트 기능)
테스트 케이스 작성 시 가독성을 높일 수 있도록 에디터에 리스트 기능을 추가하고 싶습니다.

- 번호 리스트(1. 2. 3.)
- 불렛 리스트(• 형태)

두 기능 모두 케이스 생성/편집 화면에서 동일하게 지원되면 좋겠습니다.
또한 엔터 입력 시 다음 리스트 항목이 자동으로 생성되는 기본 동작도 포함해주세요.

✔ 폴더 삭제 시 연관 문제 디버깅 요청
폴더를 지웠을 때 안에 있던 테스트 케이스들도 제대로 같이 지워지는지 확인해줘.
그리고 삭제 후 화면이 최신 리스트를 다시 불러오는 건지,
아니면 기존 데이터를 그대로 보여주는 건지도 확인해줘.

🔍 개발 과정 – 1주일 동안 진행한 실제 작업 흐름
1) 기능 정의

개발보다는 QA 경험이 큰 역할을 했던 단계.
사용자 흐름, 예외 케이스, 화면 구성 등을 먼저 텍스트로 정리했습니다.

2) Cursor에 기능을 설명

코드는 작성하지 않지만,
기능의 목적·흐름·조건을 명확히 설명하는 방식으로 AI를 제어했습니다.

3) 구현 → 테스트 → 디버깅 반복

기능이 예상과 다르게 구현되면 프롬프트를 정교하게 조정

로그 추가 요청

기능 단위로 흐름 재설명

QA식 분석으로 문제 원인 좁혀가기

4) MVP 완성 후 실사용 테스트

1주일 내 MVP 구현 완료 → 팀 내에서 실사용 테스트 진행 중

🧠 무엇을 배웠나
1) AI 개발의 핵심은 “기능을 얼마나 명확히 이해했는가”

언어·프레임워크보다
“무엇을 만들고 싶은지”를 설명하는 능력이 더 중요했습니다.

2) AI는 코드를 쓰지만, 시스템 구조와 목적은 사람이 정의한다

어떤 데이터가 필요한지

어떤 흐름인지

어떤 예외가 있는지

이런 부분은 사람이 설계해야 합니다.

3) 코드를 몰라도 시스템 이해도는 올라갈 수 있다

AI가 작성한 코드를 설명받고 흐름을 정리하면서
전체 아키텍처를 자연스럽게 이해하게 되었습니다.

📝 회고

AI를 활용하면 개발이 쉽게 끝날 것 같지만,
실제로는 설계가 모호하면 원하는 기능이 절대 나오지 않습니다.

기능이 많아질수록

AI의 맥락 손실

기존 기능 변경

연계 오류 발생

같은 문제도 자주 등장했습니다.

하지만 이 과정에서

기능을 정확히 설명하는 능력

문제를 구조적으로 이해하는 능력

QA로서 시스템을 보는 시야

가 비약적으로 성장했습니다.

AI는 개발을 대신해줄 수 있지만,
방향을 정하는 건 결국 사람이다.
